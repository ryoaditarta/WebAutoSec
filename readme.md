# Automated Web Attack Visibility & Security Triage Platform

This project is designed to automate security triage and improve visibility of web-based attacks through centralized monitoring, automated vulnerability scanning, and security telemetry visualization.

## Main Features

1. Automated web vulnerability scanning using OWASP ZAP

2. Centralized security telemetry collection and visualization

3. Automated security finding classification based on severity and confidence level

4. Real-time dashboard monitoring for attack visibility and observability

5. Containerized deployment using Docker Compose

6. Automated report generation in JSON format for security analysis

7. Detection and visualization of common web attacks such as:
   - SQL Injection
   - Cross-Site Scripting (XSS)
   - Security Misconfiguration
   - Sensitive Information Disclosure

8. Security alert pipeline for high severity findings

9. Security event parsing and processing using Python automation scripts

10. Extensible architecture for integration with:
    - Grafana
    - Elasticsearch
    - Loki
    - Kafka
    - Prometheus

## Main Objective

The main objective of this project is to simulate a lightweight SOC-oriented web application security monitoring environment capable of automatically detecting, classifying, and visualizing web application attacks.

## Use Case

This platform can help security teams:
- reduce manual vulnerability assessment effort
- improve attack visibility
- centralize web security findings
- support faster security triage and incident investigation
